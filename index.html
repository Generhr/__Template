C++ Project Template

This is a template for C++ projects. What you get:

  • Library, executable and test code separated in distinct folders
  • Use of modern CMake for building and compiling
  • External libraries installed and managed by
    • Conan Package Manager OR
    • Vcpkg Package Manager
  • Unit testing using Googletest
  • General purpose libraries: JSON, spdlog, cxxopts and fmt
  • Continuous integration testing with Github Actions and pre-commit
  • Code documentation with Doxygen and Github Pages
  • Code analysis: Cppcheck
  • Code coverage reports, including automatic upload to Codecov
  • Tooling: CCache, Clang-Format, CMake-Format, and Sanitizers

Structure

├── CMakeLists.txt
├── cmake
│ └── CMake modules
│ └── ...
├── doc
│ ├── Doxyfile
│ └── html/
├── external
│ ├── CMakesLists.txt
├── include
│ ├── pch.h
│ └── ...
├── lib
│ └── ...
├── src
│ └── Main.cpp
│ └── ...
└── test
├── CMakeLists.txt
└── ...

Headers go in include/, source code in src/, and unit tests go in test/.

Software Requirements

  • CMake v3.16+
  • GNU Makefile
  • Conan or Vcpkg
  • MSVC 2017 (or higher), G++9 (or higher), Clang++9 (or higher)
  • (Optional) Doxygen and Graphviz
  • (Optional) Cppcheck
  • (Optional, GNU or Clang only) Code Coverage: lcov and gcovr

Additional setup

GitHub

  • Give your workflows write permission in Settings/Actions/General/Workflow permissions
  • Setup GitHub Pages

Local Machine

  • (Optional) Automate unit tests with git hooks
  • (Optional) Install the various hooks listed in .pre-commit-config.yaml for use with pre-commit or delete the file if you don't intend to use it. Pre-commit can be automated with git hooks like ctest above
  • (Optional) Setup Ccache to speed up subsequent compilations. It is already built into the project but only runs if the program can be found
  • (Optional, Windows only) If you have multiple projects, consider using symlinks to use the same formatting settings (.clang-format, .clang-tidy, etc..) for all your projects with a single file. There is a batch file that will perform this process automatically but you will need to change the source directory. The first two lines are to run with admin privileges

Building

First, clone this repo:

git clone --recursive https://github.com/Onimuru/__Template
  • Documentation
cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
cmake --build . --config Debug --target doxygen
  • Code Analysis
cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
cmake --build . --config Debug --target cppcheck
  • Executable
cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
cmake --build . --config Release
cd bin/Release
./<your project name>
  • Unit testing
cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
cmake --build . --config Debug --target unit_tests
ctest -C Debug -VV
  • Code Coverage (Unix only)
cd build
cmake -DCMAKE_BUILD_TYPE=Debug -DENABLE_CODE_COVERAGE=ON ..
cmake --build . --config Debug --target coverage

For more info about CMake see here.
